# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please **do not** open a public issue.

Instead, report privately by email:

- **Email:** `pp.namias@gmail.com`
- **Subject suggestion:** `Security report: <short-summary>`

Include:

- Description of the issue
- Affected file/feature
- Reproduction steps or proof-of-concept
- Potential impact
- Suggested mitigation (if available)

## Response Expectations

- Initial acknowledgment target: within 72 hours
- Triage and severity assessment: as soon as possible
- Patch timeline depends on severity and complexity

## Scope

This policy covers:

- Website application code in this repository
- Build and CI configuration
- Dependency vulnerabilities

Out of scope:

- Issues requiring physical/local-only machine access
- Social engineering attempts
- Non-security feature requests

## Coordinated Disclosure

Please allow time for investigation and remediation before public disclosure.

Thank you for helping keep this project secure.
