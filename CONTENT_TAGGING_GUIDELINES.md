### 🎯 Overview
This document provides clear guidelines for categorizing and tagging content in the Kenneth Namias portfolio to maintain consistency, improve discoverability, and ensure a professional content organization system.

## 🏷️ TAG SYSTEM

### **Core Tag Rules**
- **Maximum 5 tags per content piece**
- **Minimum 2 tags per content piece**
- **Required: 1 Technology tag + 1 Topic tag + 1-3 Specific tags**
- **No personal names as tags**
- **No institution names as tags**
- **No single-use tags**

## ✅ CONTENT CREATION CHECKLIST

### **Before Publishing Any Content:**

#### **1. Category Assignment**
- [ ] Content fits clearly into ONE of the 5 blog categories
- [ ] Category choice is logical and consistent
- [ ] Content doesn't overlap with other categories

#### **2. Tag Selection**
- [ ] Selected 2-5 tags maximum
- [ ] Included at least 1 Technology tag
- [ ] Included at least 1 Topic tag
- [ ] All tags are from the approved list
- [ ] No redundant or overlapping tags
- [ ] No personal names or institution names

#### **3. Content Quality**
- [ ] Title is clear and descriptive
- [ ] Description is compelling and accurate
- [ ] Tags accurately reflect the content
- [ ] Category is appropriate for the content type

#### **4. SEO Optimization**
- [ ] Keywords are naturally integrated
- [ ] Tags support SEO strategy
- [ ] Content is discoverable through tags
- [ ] No keyword stuffing

## 🚫 FORBIDDEN PRACTICES

### **Never Use These as Tags:**
```yaml
Personal Names:
- Kenneth Namias
- John Kenneth Ryan Namias
- Any personal names

Institution Names:
- UCC Congressional Campus
- Company names
- School names

Generic Terms:
- Programming
- Development
- Technology
- Software
- Computer

Demo/Example Tags:
- Demo
- Example
- Test
- Sample
- Foo, Bar

Single-Use Tags:
- Framework-specific versions (React.js vs React)
- Overly specific project names
- One-off terms
```

### **Tag Redundancy to Avoid:**
```yaml
DON'T USE BOTH:
- React + React.js (use only "React")
- AI + Machine Learning (use only "AI/ML")
- Full-Stack + Full Stack Developer (use only "Full-Stack")
- DevOps + CI/CD (use only "DevOps")
- Database + PostgreSQL (use only "Database")
```

## 📊 USAGE EXAMPLES

### **Good Tagging Examples:**

#### **Technical Blog Post:**
```yaml
Title: "Building Scalable React Applications with TypeScript"
Category: Technical
Tags: ["React", "TypeScript", "Tutorial", "Best-Practices", "Full-Stack"]
✅ Perfect: 1 Tech + 1 Topic + relevant specifics
```

#### **Project Case Study:**
```yaml
Title: "AI Agent for Financial Services: Case Study"
Category: Projects  
Tags: ["AI/ML", "Case-Study", "FinTech", "Enterprise", "Automation"]
✅ Perfect: Technology + Topic + Industry + Business focus
```

#### **Career Content:**
```yaml
Title: "My Journey to Full-Stack Development"
Category: Career
Tags: ["Full-Stack", "Education", "Best-Practices"]
✅ Perfect: Clean, focused, relevant
```

### **Bad Tagging Examples:**

#### **Over-tagged Content:**
```yaml
❌ BAD:
Tags: ["React", "React.js", "JavaScript", "TypeScript", "Frontend", "Full-Stack", "Programming", "Development", "Tutorial", "Guide"]
Problems: Too many tags, redundancy, generic terms
```

#### **Personal Name Pollution:**
```yaml
❌ BAD:
Tags: ["Kenneth Namias", "John Kenneth Ryan Namias", "UCC Congressional Campus", "Personal Journey"]
Problems: Personal names, institution names, not discoverable
```

#### **Generic and Useless:**
```yaml
❌ BAD:
Tags: ["Technology", "Software", "Computer", "Development", "Programming"]
Problems: Too generic, no specific value, poor discoverability
```
